## 第一章 绪论

### 1.1 计算机网络的基本概念

**计算机网络**：一些相互连接的、以共享资源为目的的、自治的计算机集合。

**两大基本功能**：
- **连通性**：使上网用户之间都可以交换信息
- **共享**：资源共享（信息共享、软件共享、硬件共享）

**基本组成**：
- 硬件系统：主机、终端、网络连接设备（网卡、Modem、中继器、集线器、网桥、交换机、路由器）
- 软件系统：网络操作系统、网络协议、网络应用软件

### 1.2 计算机网络的发展

| 阶段 | 特点 |
|------|------|
| 第一阶段 | 以计算机为中心的联机终端系统（20世纪60年代） |
| 第二阶段 | 主机-主机互连阶段（ARPANET, 60年代后期） |
| 第三阶段 | 以通信子网为中心的网络互连 |
| 第四阶段 | 统一网络体系结构、标准化协议（TCP/IP） |

**Internet发展**：
- 1969年：ARPANET实验性运行，4个节点
- 1983年：TCP/IP协议成为ARPANET标准协议（因特网诞生时间）
- 1990年：ARPANET退役，NSFNET接管并改名为Internet
- 1989年：Tim Berners-Lee提出WWW（HTTP + HTML）

**internet vs Internet**：
- **internet**（互联网）：通用名词，泛指由多个计算机网络互连而成的网络
- **Internet**（因特网）：专用名词，指当前全球最大的、开放的、采用TCP/IP协议族的特定计算机网络

### 1.3 因特网的组成

**边缘部分**：由所有连接在因特网上的主机（端系统）组成，用户直接使用

**两种通信方式**：
- **客户-服务器方式（C/S）**：客户是服务请求方，服务器是服务提供方
  - 客户程序：主动发起通信，必须知道服务器地址
  - 服务器程序：被动等待，同时处理多个客户请求，不需要知道客户地址
- **对等方式（P2P）**：每个主机既是客户又是服务器

**核心部分**：由大量网络和路由器组成，为边缘部分提供连通性服务。最核心功能是**分组交换**。

### 1.4 三种交换方式

| 比较 | 电路交换 | 报文交换 | 分组交换 |
|------|---------|---------|---------|
| 连接建立 | 需要 | 不需要 | 不需要 |
| 传输方式 | 比特流直达 | 存储转发 | 存储转发 |
| 信道占用 | 独占 | 共享 | 共享 |
| 效率 | 低（突发数据） | 中 | 高 |

**电路交换**：三个阶段——建立连接 → 通信 → 释放连接。面向连接，独占信道。

**分组交换**：
- 将报文划分成较短的、固定长度的数据段
- 每个数据段添加首部构成**分组**
- 首部包含地址、差错控制等信息
- 优点：高效、灵活、迅速、可靠
- 问题：排队时延、首部开销

### 1.5 计算机网络的分类

**按作用范围**：
- **PAN**：个人区域网
- **LAN**：局域网（10~1000Mbps, 0.1~10km）
- **MAN**：城域网
- **WAN**：广域网

**按传输技术**：
- 广播网络（共享信道）
- 点对点网络

### 1.6 计算机网络的性能指标

| 指标 | 定义 | 单位/公式 |
|------|------|----------|
| **速率** | 数据率/比特率 | b/s, kb/s, Mb/s, Gb/s |
| **带宽** | 数字信道能传送的最高数据率 | b/s |
| **吞吐量** | 单位时间内通过某网络的实际数据量 | b/s |
| **发送时延** | 数据块从结点进入传输媒体所需时间 | 数据块长度/信道带宽 |
| **传播时延** | 电磁波在信道中传播所需时间 | 信道长度/传播速率 |
| **处理时延** | 交换结点存储转发处理时间 | — |
| **排队时延** | 结点缓存队列中排队时间 | — |
| **总时延** | = 发送时延 + 传播时延 + 处理时延 + 排队时延 | — |
| **时延带宽积** | = 传播时延 × 带宽 | bit |
| **往返时间RTT** | 发送方发送数据到收到确认的总时间 | — |
| **利用率** | 信道被利用的时间百分比 | D = D₀/(1-U) |

> **重要概念**：提高链路带宽减小的是数据的**发送时延**，而不是比特在链路上的**传播速率**。

### 1.7 网络体系结构

**网络协议的三要素**：
1. **语法**：数据与控制信息的结构或格式
2. **语义**：需要发出何种控制信息，完成何种动作
3. **同步**：事件实现顺序的详细说明

**分层思想的优势**：
- 各层之间独立，仅需知道层间接口
- 灵活性好，层内改变不影响上下层
- 结构上可分割，易于实现和维护
- 促进标准化

**各层需要完成的主要功能**：
- 差错控制、流量控制、分段和重装、复用和分用、连接建立和释放

**重要概念**：
- **实体**：表示任何可发送或接收信息的硬件或软件进程
- **协议**：控制两个对等实体通信的规则集合（**水平方向**）
- **服务**：下层向上层通过层间接口提供的功能（**垂直方向**）
- **SAP**（服务访问点）：相邻两层实体交互的地方
- **体系结构**：计算机网络的各层及其协议的集合（抽象）
- **实现**：遵循体系结构用硬件或软件完成功能（具体）

### 1.8 OSI参考模型（七层）

| 层次 | 名称 | 主要功能 |
|------|------|---------|
| 7 | 应用层 | HTTP, FTP, QQ等网络应用 |
| 6 | 表示层 | 信息语法和语义，数据结构抽象定义 |
| 5 | 会话层 | 对话控制、令牌管理、同步（检查点） |
| 4 | 传输层 | 端到端可靠传输，差错控制，流量控制 |
| 3 | 网络层 | 路由选择、转发、拥塞控制、QoS |
| 2 | 数据链路层 | 成帧、物理寻址、流量控制、差错控制 |
| 1 | 物理层 | 原始比特流传输、机械/电气/功能/过程特性 |

> OSI在市场化方面失败了（协议复杂、效率低、周期长、层次划分欠合理）。

### 1.9 TCP/IP参考模型（四层）

| 层次 | 名称 | 典型协议 |
|------|------|---------|
| 4 | 应用层 | HTTP, SMTP, DNS, FTP |
| 3 | 传输层 | TCP, UDP |
| 2 | 网际层 | IP |
| 1 | 网络接口层 | （无具体内容） |

> TCP/IP是**事实上的国际标准（de facto）**。

### 1.10 五层协议体系结构（教学常用）

| 层次 | 名称 | PDU名称 | 关键功能 |
|------|------|---------|---------|
| 5 | 应用层 | 应用层PDU | 为用户应用进程提供网络服务 |
| 4 | 传输层 | 报文段(TCP)/用户数据报(UDP) | 端到端可靠/不可靠传输 |
| 3 | 网络层 | IP数据报（分组） | 路由选择与转发 |
| 2 | 数据链路层 | 帧 | 相邻结点间可靠传输 |
| 1 | 物理层 | 比特流 | 物理媒体上的比特传输 |

**数据封装过程**（发送端）：应用层 → 传输层加首部 → 网络层加首部 → 链路层加首部和尾部 → 物理层转比特流
**数据解封过程**（接收端）：物理层 → 链路层剥首尾 → 网络层剥首部 → 传输层剥首部 → 应用层 → 应用进程

## 第二章 物理层

### 2.1 物理层基本概念

物理层主要任务：将原始比特流从一结点传输到相邻的另一结点。

**四大特性**：
- **机械特性**：接线器形状、尺寸、引线数目、排列
- **电气特性**：接口电缆各条线上电压的范围
- **功能特性**：某电平电压表示的意义
- **过程特性**：各种可能事件的出现顺序

### 2.2 数据通信基础

**相关术语**：
- **数据**：运送消息的实体
- **信号**：数据的电气的或电磁的表现
- **模拟信号**：参数取值连续
- **数字信号**：参数取值离散
- **码元**：代表不同离散数值的基本波形

**通信方式**：
- **单工**：只能单向通信
- **半双工**：双方可发送但不能同时
- **全双工**：双方可同时发送和接收

**基带信号与通带信号**：
- **基带信号**：直接来自信源的信号（含低频/直流成分）
- **通带信号**：基带信号经过载波调制后搬移到较高频段

### 2.3 调制方法

调制解调器（Modem）= 调制器 + 解调器

**基本二元制调制方法**（载波信号：A·sin(2πft + φ)）：

| 方法 | 变化参数 | 简称 |
|------|---------|------|
| 调幅 | 振幅 A | AM |
| 调频 | 频率 f | FM |
| 调相 | 相位 φ | PM |

**正交振幅调制（QAM）**：振幅和相位结合，一个码元可传输更多比特。

**码元与比特率的关系**：若一个码元表示n比特，则比特率 = 波特率 × n

### 2.4 信道的极限容量

**香农定理**（有噪声信道的极限速率）：

$$C = W \cdot \log_2(1 + S/N) \quad \text{(b/s)}$$

其中：
- W：信道带宽（Hz）
- S/N：信噪比
- (S/N)dB = 10 × log₁₀(S/N)

**香农公式表明**：
- 带宽越大或信噪比越大，极限传输速率越高
- 只要传输速率 < 极限速率，理论上可实现无差错传输
- 实际能达到的速率远低于香农极限

### 2.5 传输媒体

#### 导向传输媒体

| 类型 | 特点 | 应用 |
|------|------|------|
| **UTP（无屏蔽双绞线）** | 便宜、易安装、有辐射、易被窃听 | 室内网络连接 |
| **STP（屏蔽双绞线）** | 抗干扰强 | 特殊环境 |
| **同轴电缆（50Ω）** | 频带宽、损耗低、抗干扰强 | 计算机联网（室内细缆/室外粗缆） |
| **同轴电缆（75Ω）** | 宽带、多路复用 | 有线电视 |
| **光纤** | 频带宽、损耗极低、抗干扰、轻便 | 长途干线、接入网 |

**光纤原理**：
- 纤芯（高折射率）+ 包层（低折射率）
- 光线在纤芯中通过全反射传输
- 多模光纤 vs 单模光纤

#### 非导向传输媒体

- 短波通信（电离层反射，质量较差）
- 微波通信（直线传播，地面接力 + 卫星通信）
- 低轨道卫星（如铱星系统、Starlink星链）

### 2.6 信道复用技术

**复用（Multiplexing）**：多个用户共享同一信道。

| 复用方式 | 原理 | 特点 |
|----------|------|------|
| **FDM（频分复用）** | 不同用户占用不同频带 | 同时不同频 |
| **TDM（时分复用）** | 不同用户占用不同时隙 | 同频不同时，可能浪费资源 |
| **STDM（统计时分复用）** | 动态分配时隙 | 提高利用率 |
| **WDM（波分复用）** | 光的频分复用 | 光纤中广泛应用 |
| **CDM（码分复用）** | 不同编码区分用户 | 抗干扰能力强 |

### 2.7 宽带接入技术

| 技术 | 特点 |
|------|------|
| **xDSL** | 用数字技术改造模拟电话线，0~4kHz留电话，高端频谱上网 |
| **ADSL** | 非对称（下行>>上行），DMT调制技术，下行可达24Mbps |
| **HFC** | 光纤同轴混合网，基于CATV改造 |
| **FTTx** | 光纤到户/大楼/路边（FTTH/FTTB/FTTC） |
| **PLC** | 电力线通信，OFDM技术 |

> 趋势：**光进铜退** —— 光纤最终将取代铜线。

## 第三章 数据链路层

### 3.1 基本概念

**链路（link）**：一条无源的点到点物理线路段，中间无交换结点。

**数据链路（data link）**：链路 + 协议。通过数据链路层协议，在不可靠物理链路上实现可靠的数据帧传输。

**数据链路层传输单位**：**帧（frame）**

**两种信道**：
- **点对点信道**：一对一的通信方式
- **广播信道**：一对多的广播通信方式

### 3.2 三个基本问题

#### （1）封装成帧

在一段数据前后分别添加**首部**和**尾部**，构成帧。首部和尾部的重要作用是**帧定界**。

帧长 ≤ MTU（最大传输单元）

#### （2）透明传输

若数据部分出现了与帧定界符相同的比特组合，则需要转义处理。

方法：
- **字节填充（字符填充）**：在控制字符（SOH/EOT）前插入ESC转义字符
- **比特填充**：连续5个1后插入一个0

#### （3）差错检测

- **随机差错**：独立性，与前后码元无关
- **突发差错**：相邻多个数据位出错
- **误码率（BER）**：传输错误比特占总传输比特的比例

**差错控制方式**：
- **ARQ（反馈纠错）**：发现差错后请求重发（适用于双工通信）
- **FEC（前向纠错）**：接收端能发现并纠正错误（适用于单工/实时通信）
- **混合纠错**：少量自动纠正，超出能力请求重发

**检错编码**：
- **奇偶校验码**：简单，检错能力差
- **CRC（循环冗余编码）**：实现容易、检错能力强、广泛使用
- **海明码**：可纠错和检错，但实现复杂

### 3.3 PPP协议

PPP（Point-to-Point Protocol）是目前使用最多的数据链路层协议。

**PPP协议满足的需求**：简单、封装成帧、透明性、多种网络层协议、多种链路类型、差错检测、MTU等。

**PPP协议不需要的功能**：纠错（由TCP负责）、流量控制（由TCP负责）、序号、多点线路、半双工/单工。

### 3.4 广播信道与局域网

**局域网特点**：高数据率(10~1000Mbps)、短距离(0.1~10km)、低误码率。

**局域网拓扑**：星形、总线、环形、树形

**决定局域网性能的三要素**：网络拓扑、传输介质、介质访问控制方法。

#### 介质访问控制（MAC）

- **静态分配**：FDM、TDM — 不适合站点数变化的场景
- **动态分配**：
  - **受控接入**（如令牌环网）
  - **随机接入**（如以太网）

#### 局域网参考模型（IEEE 802）

分为两个子层：
- **MAC子层**（介质访问控制）：成帧/拆帧、比特差错检测、寻址、竞争处理（与介质相关）
- **LLC子层**（逻辑链路控制）：建立/释放逻辑连接、提供高层接口、差错控制、帧序号（与介质无关）

> 目前TCP/IP体系常直接用以太网V2标准，LLC子层作用已不大。

### 3.5 以太网

#### 标准

- **DIX Ethernet V2**：世界上第一个局域网产品规约
- **IEEE 802.3**：与DIX V2只有很小差别

#### CSMA/CD协议

以太网的核心是**CSMA/CD（载波监听多点接入/碰撞检测）**。

**CSMA协议分类**：
| 类型 | 特点 |
|------|------|
| 1-坚持CSMA | 监听空闲则立即发送；低负载时效率高 |
| 非坚持CSMA | 监听忙则随机等待再监听；高负载时效率高 |
| p-坚持CSMA | 以概率p发送 |

**CSMA/CD = CSMA + 碰撞检测（Collision Detection）**

**二进制指数退避算法**：
- 基本退避时间 = 2τ（争用期）
- k = Min[重传次数, 10]
- 从[0, 2ᵏ-1]中随机取r，退避时间 = r × 基本退避时间
- 16次重传仍失败则丢弃帧

#### 曼彻斯特编码

以太网使用曼彻斯特编码：每比特中间都有一个电平跳变，自带同步信号。

#### MAC地址（硬件地址）

- 共6字节（48位）
- 前3字节：厂家标识（IEEE RA分配）
- 后3字节：厂家自行分配
- I/G位：0=单地址，1=组地址
- U/L位：1=全球管理，0=本地管理

#### MAC帧格式（DIX Ethernet V2）

```
目的地址(6B) | 源地址(6B) | 类型(2B) | 数据(46~1500B) | FCS(4B)
```

- 类型字段：标识上层协议（如IP=0x0800）
- 数据字段：46~1500字节
- 有效帧长度：**64~1518字节**
- FCS：4字节帧检验序列（CRC校验）

#### 发送/接收类型

- **单播（unicast）**：一对一
- **广播（broadcast）**：一对全体
- **多播（multicast）**：一对多

### 3.6 扩展的以太网

#### 集线器（Hub）
- 工作在物理层
- 逻辑上仍是总线网，仍使用CSMA/CD
- 所有端口在同一**冲突域（碰撞域）**内
- 不能隔离冲突域

#### 网桥（Bridge）
- 工作在数据链路层
- 根据MAC帧目的地址对帧进行转发/过滤
- 可隔离冲突域（每个端口一个冲突域）
- **透明网桥**：即插即用，采用自学习算法
- 使用**生成树算法**避免帧在网络中不断兜圈子
- 自学习：记录源地址、接口和时间

**网桥自学习与转发步骤**：
1. 收到帧，先自学习源地址→接口映射
2. 查找目的地址：有则按接口转发；无则向所有接口扩散
3. 若目的接口=入接口，丢弃帧

#### 以太网交换机（Switch）
- 实质上是**多接口网桥**
- 工作在数据链路层
- 每端口独占带宽，全双工工作
- 可同时连通多对接口，无碰撞传输
- N对接口的总容量 = N × 接口带宽（最大优点）

#### 虚拟局域网（VLAN）
- 由一些局域网网段构成的**与物理位置无关的逻辑组**
- 限制广播信息，避免"广播风暴"

### 3.7 高速以太网

| 类型 | 速率 | 传输介质 | 特点 |
|------|------|---------|------|
| 100BASE-T | 100Mbps | 双绞线/光纤 | 仍使用CSMA/CD |
| 吉比特以太网(GE) | 1Gbps | 光纤/UTP | 全双工不用CSMA/CD |
| 10吉比特以太网(10GE) | 10Gbps | 仅光纤 | 仅全双工，不用CSMA/CD |

### 3.8 无线局域网（WLAN）

- 标准：IEEE 802.11 a/b/g/n/ax (WiFi6: 9.6Gbps)
- **BSS**（基本服务集）：一个AP + 若干移动站
- **AP**（接入点）：作用和网桥相当，分配SSID和信道
- **ESS**（扩展服务集）：通过DS连接多个BSS
- **关联（association）**：移动站加入BSS需先选择AP并建立关联

**无线局域网特殊问题**：
- **隐蔽站问题**：A和C都以为B空闲，同时发数据给B导致碰撞
- **暴露站问题**：B向A发送，C听到了但C向D发送并不影响B→A

**CSMA/CA（碰撞避免）**：
- 不是传统的CSMA/CD（无线难以检测碰撞）
- MACA协议：发送前先发RTS预约，接收方回复CTS确认预约
- 周围监听到RTS/CTS的站点避免发送

## 第四章 网络层

### 4.1 网络层概述

网络层解决的核心问题：**网络的互连**。

TCP/IP体系网络层 = **IP层**，属于通信子网最高层。

**IP层特点**：
- 提供**无连接**的、**尽力而为（Best Effort）**的分组传输
- 不提供可靠性保障（可靠性由传输层TCP实现）
- IP协议是**点到点**的
- IP层隔离了各种物理网络的差异

### 4.2 IP地址

**IP地址**：32位，全球唯一，由ICANN统一分配。采用**点分十进制表示法**（如192.2.3.4）。

**IP地址层次结构**：网络号(网络前缀) + 主机号

#### 分类IP地址

| 类别 | 特征位 | 网络号位数 | 主机号位数 | 范围 |
|------|--------|-----------|-----------|------|
| A类 | 0 | 8 | 24 | 1.0.0.0 ~ 126.0.0.0 |
| B类 | 10 | 16 | 16 | 128.0.0.0 ~ 191.255.0.0 |
| C类 | 110 | 24 | 8 | 192.0.0.0 ~ 223.255.255.0 |
| D类 | 1110 | — | — | 组播地址 |
| E类 | 11110 | — | — | 保留 |

#### 特殊IP地址

| 地址 | 含义 |
|------|------|
| 主机号全1 | 直接广播地址（向指定网络广播） |
| 255.255.255.255 | 有限广播地址（本网络广播） |
| 网络号全0 | 本网络 |
| 127.x.x.x | 回送地址（本地测试，如127.0.0.1） |

#### 私有IP地址（保留地址段）

| 范围 | CIDR |
|------|------|
| 10.0.0.0 ~ 10.255.255.255 | 10.0.0.0/8 |
| 172.16.0.0 ~ 172.31.255.255 | 172.16.0.0/12 |
| 192.168.0.0 ~ 192.168.255.255 | 192.168.0.0/16 |

#### 子网掩码

- 定义IP地址中网络前缀的比特数长度
- 高位为1（网络位），低位为0（主机位）
- 网络地址 = IP地址 & 子网掩码
- 表示法：点分十进制（255.255.0.0）或 CIDR（/16）

**路由器/多穴主机**：每个网络连接对应一个IP地址，有多少网络连接就有多少IP地址。

### 4.3 地址解析

**地址解析**：IP地址 ↔ 物理地址之间的映射。

#### ARP（地址解析协议）

**工作过程**：
1. 主机A查看ARP缓存中有无主机B的IP地址映射
2. 若有，直接使用；若无，广播发送ARP请求
3. 主机B收到请求，单播回复ARP响应（包含自己的硬件地址）
4. 主机A将映射存入ARP缓存

> ARP只解决**同一局域网**内IP地址→硬件地址的映射。不同网络需通过路由器转发。

#### RARP（逆地址解析协议）

使只知硬件地址的主机获取IP地址（主要用于无盘工作站，已很少使用）。

### 4.4 IP数据报格式

| 字段 | 位 | 说明 |
|------|-----|------|
| 版本 | 4 | IPv4 = 4 |
| 首部长度 | 4 | 以4字节为单位，最大60字节 |
| 区分服务 | 8 | 用于DiffServ |
| 总长度 | 16 | 首部+数据，最大65535字节 |
| 标识 | 16 | 分片标识，同一分组的分片有相同ID |
| 标志 | 3 | MF=1还有分片，DF=0允许分片 |
| 片偏移 | 12 | 以8字节为单位，分片在原始分组中的位置 |
| TTL | 8 | 生存时间，每经过路由器减1，为0时丢弃 |
| 协议 | 8 | 上层协议类型（TCP=6, UDP=17） |
| 首部检验和 | 16 | 仅检验首部，不检验数据 |
| 源地址 | 32 | 发送方IP地址 |
| 目的地址 | 32 | 接收方IP地址 |

### 4.5 IP分组的分片与重组

- **MTU（最大传输单元）**：物理网络对帧大小的限制
- **分片**：当新网络MTU小于原网络MTU时，在路由器上进行
- **重组**：**仅在目的主机上进行**

**分片相关字段**：
- **标识（ID）**：同一原始分组的所有分片ID相同
- **MF**：=1表示后面还有分片，=0表示最后一个分片
- **DF**：=0时才允许分片
- **片偏移**：以8字节为单位，表示该片在原始分组数据中的位置

### 4.6 路由与转发

**选路（Routing）**：根据目的地址寻找从源到目的的通路。

**分组转发**：采用**hop-by-hop**方式，每经过一个路由器都需要选路。

**直接转发 vs 间接转发**：
- 直接转发：目的结点在同一物理网络中，不需要经过其他路由器
- 间接转发：目的结点不在同一物理网络，需选择下一跳路由器

#### 路由表

由（N, R）二元组构成：
- N：目的网络地址
- R：下一跳路由器地址

**最长前缀匹配**：当多个表项匹配时，选择网络前缀最长的。

**默认路由**：当路由表中无匹配项时，使用预定义的默认路由转发。

#### 路由协议分类

| 类型 | 说明 | 典型协议 |
|------|------|---------|
| **IGP**（内部网关协议） | 自治系统AS内部使用 | RIP, OSPF |
| **EGP**（外部网关协议） | 不同AS之间使用 | BGP-4 |

**自治系统（AS）**：由一个机构统一管理的网络和路由器的集合。

#### 距离向量法（RIP）

- 各路由器将已知路由信息通知相邻路由器
- 距离向量 = {network, cost}
- **RIP**：采用距离向量算法，每30秒广播一次，最大跳数=15 hop
- 不支持VLSM（RIPv2支持）

#### 链路状态法（OSPF）

- 每个路由器将链路状态发布给全网
- 发布**LSP（链路状态分组）**：路由器标识、相邻路由器、链路费用
- 利用Dijkstra算法计算最短路径
- **OSPF**：采用链路状态法 + SPF算法，支持VLSM、多条等价通路
- 层次化结构：分Area，骨干区域为Area 0

#### 通路向量法（BGP）

- **BGP-4**：域间路由协议
- 采用通路向量选路（path vector）
- 支持基于策略的路由
- 使用TCP保证路由器间信息交换的可靠性

## 第五章 传输层

### 5.1 传输层概述

**传输层 vs 网络层**：
- 传输层：为**应用进程之间**提供逻辑通信（端到端）
- 网络层：为**主机之间**提供逻辑通信（点到点）

**引入传输层的原因**：
1. 消除网络层的不可靠性，提供可靠的端到端传输
2. 提供与实际网络无关的信息传输（屏蔽通信子网差异）

**TCP/IP体系中的传输层**：
- **UDP**：用户数据报协议，无连接、不可靠
- **TCP**：传输控制协议，面向连接、可靠

### 5.2 端口

**端口（Port）**：传输层服务访问点（TSAP），16位，最多64K个。

**套接字（Socket）** = IP地址 + 端口号

**端口号分类**：

| 范围 | 类型 | 说明 |
|------|------|------|
| 0~1023 | 熟知端口（保留端口） | ICANN统一分配 |
| 1024~49151 | 登记端口号 | 应用程序使用 |
| 49152~65535 | 客户端口号（短暂端口） | 客户进程动态分配 |

**常用熟知端口号**：

| UDP | 服务 | TCP | 服务 |
|-----|------|-----|------|
| 53 | DNS | 20 | FTP数据 |
| 67 | BOOTPS | 21 | FTP控制 |
| 69 | TFTP | 23 | Telnet |
| 161 | SNMP | 80 | HTTP |
| — | — | 25 | SMTP |
| — | — | 110 | POP3 |

**端口分配方式**：
- 全局统一分配（保留端口供服务器进程）
- 本地动态绑定（自由端口供客户进程）
- 客户-服务器模式：客户使用自由端口联系服务器的保留端口

### 5.3 UDP（用户数据报协议）

**UDP特点**：
- **无连接**：发送前不需建立连接
- **不可靠**：不保证交付，不使用拥塞控制
- **面向报文**：对应用层报文既不合并也不拆分
- 支持一对一、一对多、多对一、多对多
- **首部开销小**：仅8字节
- 效率高，适合实时应用（多媒体通信）

**UDP数据报格式**（首部8字节）：

```
源端口(2B) | 目的端口(2B) | 长度(2B) | 校验和(2B) | 数据
```

**UDP校验和**：
- 可选功能（禁止可提高效率）
- 计算覆盖：UDP伪报头 + UDP数据报
- 伪报头：源IP + 目的IP + 协议号(UDP=17) + UDP长度

### 5.4 TCP（传输控制协议）

**TCP特点**：
- **面向连接**：使用前需建立连接
- **点对点**：每条连接只有两个端点
- **可靠交付**：确认 + 重传 + 滑动窗口 + 流量控制 + 拥塞控制
- **全双工通信**
- **面向字节流**：数据无结构，字节流形式传输

**TCP连接**：
- 端点：套接字 socket = (IP地址, 端口号)
- TCP连接 ::= {socket₁, socket₂} = {(IP₁:port₁), (IP₂:port₂)}

#### TCP报文段格式

```
源端口(2B) | 目的端口(2B)
序号(4B)
确认号(4B)
数据偏移(4b) | 保留(6b) | 码元比特(6b) | 窗口(2B)
校验和(2B) | 紧急指针(2B)
选项(可变) | 填充
```

**码元比特（6位）**：

| 位 | 名称 | 含义 |
|-----|------|------|
| URG | 紧急 | URG=1时紧急指针有效 |
| ACK | 确认 | ACK=1时确认号字段有效 |
| PSH | 推送 | PSH=1时尽快交付给应用进程 |
| RST | 复位 | RST=1时释放连接重新建立 |
| SYN | 同步 | SYN=1表示连接请求或接受 |
| FIN | 终止 | FIN=1释放连接 |

#### TCP可靠性机制

**确认与重传**：
- **累积确认**：确认的是连续正确收到的字节流中最后一个字节+1
- **超时重传**：发送后启动定时器，超时未收到确认则重传
- TCP使用自适应重传算法动态调整定时器值

#### 滑动窗口

**目的**：提高传输效率、进行流量控制。

- 允许发送方在确认前发送多个报文段
- 发送窗口大小 = 允许未确认的最大数据量
- 接收窗口只接收窗口内的数据
- 传输通道容量 = 通道带宽 × 通道往返时延(RTT)

**TCP流量控制**：通过可变窗口大小实现
- 确认报文中包含窗口通告（接收方还能接收的数据量）
- 窗口值增大 → 扩大发送窗口
- 窗口值减小 → 缩小发送窗口
- 零通告 → 停止发送

#### TCP连接管理

**三次握手建立连接**：

```
1. 客户 → 服务器：SYN (seq=x)
2. 服务器 → 客户：SYN + ACK (seq=y, ack=x+1)
3. 客户 → 服务器：ACK (ack=y+1)
```

**四次挥手释放连接**：

```
1. 客户 → 服务器：FIN (seq=x)
2. 服务器 → 客户：ACK (ack=x+1)
3. 服务器 → 客户：FIN + ACK (seq=y, ack=x+1)
4. 客户 → 服务器：ACK (ack=y+1)
```

#### TCP拥塞控制

**拥塞**：短时间内大量分组竞争同一资源，导致队列溢出，性能下降。

**拥塞窗口（Congestion Window）**：

$$AWin = \min(Rcvr\\_Adv, Cong\\_Win)$$

- Rcvr_Adv：接收方通告窗口
- Cong_Win：拥塞窗口

**拥塞控制策略**：
- **拥塞发生时**：加速递减策略 → 拥塞窗口减半（最小减至1），重传定时器时限加倍
- **拥塞恢复**：**慢启动算法** → 拥塞窗口从1开始，每收到确认+1，指数增长
- **拥塞避免**：窗口达到上次拥塞的一半时进入拥塞避免，线性增长（所有报文段确认后+1）

## 第六章 应用层

### 6.1 域名系统（DNS）

**功能**：实现域名到IP地址的映射。

**DNS特征**：
- 基于Client/Server模式
- 基于UDP协议（端口53）
- 分布式数据库系统
- 多层次的、基于域的命名系统

**域名结构**：层次树状结构，各分量用点隔开。
→ 如 `www.bupt.edu.cn`（三级域名.二级域名.顶级域名）

**域名解析过程**：
1. 客户将域名放在DNS报文中，向本地域名服务器发出请求
2. 本地服务器可解析直接回复；否则向上级递归查询
3. 各级域名服务器使用**高速缓存**优化查询效率

### 6.2 DHCP（动态主机配置协议）

**功能**：即插即用连网，自动获取IP地址等配置。

**配置项目**：IP地址、子网掩码、默认路由器IP、域名服务器IP。

**工作过程**（基于UDP）：
1. 客户广播发送 DHCPDISCOVER（发现报文）
2. DHCP服务器回复 DHCPOFFER（提供报文）
3. 客户选择服务器，发送 DHCPREQUEST（请求报文）
4. 服务器发送 DHCPACK（确认报文），进入绑定状态
5. 租用期过半(T=0.5T)时，客户请求续租
6. 服务器同意发DHCPACK续租，不同意发DHCPNACK
7. 客户可随时发DHCPRELEASE释放

**租用期（lease period）**：IP地址的临时使用期限。

**DHCP中继代理**：减少DHCP服务器数量，负责跨网络转发DHCP报文。

### 6.3 WWW（万维网）

**Web系统构成**：
- Web浏览器
- Web服务器
- HTTP协议（超文本传输协议）
- HTML语言（超文本标记语言）

**URL格式**：`<协议>://<主机>:<端口>/<路径>`
→ 如 `http://www.bupt.edu.cn/index.html`（HTTP默认端口80可省略）

**HTTP协议**：
- 应用层协议，使用TCP连接进行可靠传输
- **无状态**：每次请求独立，没有上下文对应关系
- HTTP 1.1支持持久连接（一个TCP连接可传多个请求/响应）

**HTTP报文**：
- **请求报文** = 请求行 + 头域 + 实体
- **响应报文** = 状态行 + 头域 + 实体

**常见HTTP状态码**：

| 状态码 | 说明 |
|--------|------|
| 200 | OK，请求成功 |
| 400 | Bad Request，请求语法错误 |
| 401 | Unauthorized，未授权 |
| 403 | Forbidden，拒绝服务 |
| 404 | Not Found，资源不存在 |
| 500 | Internal Server Error，服务器内部错误 |
| 503 | Server Unavailable，服务器不可用 |

**浏览器访问网页过程**：
1. 浏览器分析URL
2. DNS解析出服务器IP地址
3. 浏览器与服务器建立TCP连接
4. 发送HTTP GET请求
5. 服务器返回HTML文件
6. 释放TCP连接
7. 浏览器解析HTML并显示，自动请求引用的其他资源（图片、CSS、JS等）

### 6.4 电子邮件

**三大构件**：
- **用户代理（UA）**：用户与邮件系统接口
- **邮件服务器**：发送和接收邮件
- **协议**：SMTP（发送）、POP3/IMAP（接收）

**邮件地址格式**：`收信人信箱名@邮件主机域名`

**SMTP（简单邮件传输协议）**：
- 用于发送邮件
- 端口：25

**POP3（邮局协议）**：
- 从邮件服务器收取邮件到本地
- 端口：110

**IMAP（交互式邮件存取协议）**：
- 客户端操作反馈到服务器
- 维护服务器端邮件目录
- 端口：143

### 6.5 搜索引擎

**分类**：
- **全文搜索引擎**：基于关键字搜索（百度、Google），信息量大、更新快
- **目录搜索引擎**：人工分类整理（如早期的Yahoo），信息准确但量少

**工作原理**：
1. 网络爬虫自动抓取网页
2. 建立索引数据库
3. 用户搜索时匹配关键词并按相关度排序

**PageRank**：Google排序算法，根据网站的外部/内部链接数量和质量衡量网站价值。

**SEO（搜索引擎优化）**：通过优化网页提高搜索引擎排名。

**robots.txt**：限制搜索引擎爬虫访问的文件（放在网站根目录）。

|------|------|
| 发送时延 = 数据块长度 / 信道带宽 | 数据发送所需时间 |
| 传播时延 = 信道长度 / 传播速率 | 电磁波在信道中传播时间 |
| 总时延 = 发送时延 + 传播时延 + 处理时延 + 排队时延 | 数据经历的总时延 |
| 时延带宽积 = 传播时延 × 带宽 | 以比特为单位的链路长度 |
| D = D₀ / (1 - U) | 利用率U对时延D的影响 |
| C = W × log₂(1 + S/N) | 香农定理（信道极限速率） |
| AWin = min(Rcvr_Adv, Cong_Win) | TCP发送窗口大小 |
| 网络地址 = IP地址 & 子网掩码 | 获取网络前缀 |
